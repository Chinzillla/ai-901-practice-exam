# Module 7: Plan and Prepare to Develop AI Solutions on Azure

## Unit 4: Foundry Tools

### Source

Microsoft Learn: [Foundry Tools](https://learn.microsoft.com/en-us/training/modules/prepare-azure-ai-development/3-azure-ai-services)

### Unit Overview

This unit introduces Foundry Tools as prebuilt APIs and models for common AI tasks. Learners review Azure Language, Azure Speech, Azure Translator, Azure Document Intelligence, and Azure Content Understanding.

### Learning Objectives

- Define Foundry Tools.
- Identify common prebuilt AI tools.
- Match Foundry Tools to AI workload scenarios.
- Explain why prebuilt tools can be more predictable or cost-effective than agent-only approaches.

### Key Terms

| Term | Definition |
| --- | --- |
| Foundry Tools | Prebuilt APIs and models in Microsoft Foundry for common AI tasks. |
| Azure Language | Tooling for text analysis, language understanding, and question answering. |
| Azure Speech | Tooling for speech-to-text and text-to-speech. |
| Azure Translator | Tooling for translating text between languages. |
| Azure Document Intelligence | Tooling for extracting fields from documents. |
| Azure Content Understanding | Multimodal content analysis for documents, images, video, and audio. |

### Lesson Flow

#### 1. Why Use Prebuilt Tools

Explain that not every AI task needs a custom generative AI agent. Prebuilt services can be predictable and cost-effective.

#### 2. Tool Overview

Review Language, Speech, Translator, Document Intelligence, and Content Understanding.

#### 3. Connecting to Tools

Explain that applications connect to tool-specific endpoints using keys or token-based authentication.

### Suggested Activities

- Learners match requirements to Foundry Tools.
- Learners compare when to use a prebuilt tool versus a custom agent.

### Checks for Understanding

1. What are Foundry Tools?
2. Which tool supports translation?
3. Which tool extracts fields from invoices?
4. Which tool handles speech-to-text?
