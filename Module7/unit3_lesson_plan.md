# Module 7: Plan and Prepare to Develop AI Solutions on Azure

## Unit 3: Microsoft Foundry

### Source

Microsoft Learn: [Microsoft Foundry](https://learn.microsoft.com/en-us/training/modules/prepare-azure-ai-development/4-azure-ai-foundry)

### Unit Overview

This unit introduces Microsoft Foundry as a platform for AI development on Azure. Learners explore Foundry resources, projects, models, agents, tools, knowledge, the Foundry portal, and the Foundry SDK.

### Learning Objectives

- Define Microsoft Foundry.
- Explain Foundry resources and projects.
- Identify assets managed in a Foundry project.
- Describe the Foundry portal and Foundry SDK.
- Recognize how projects centralize AI solution development.

### Key Terms

| Term | Definition |
| --- | --- |
| Microsoft Foundry | Azure platform for AI development. |
| Foundry portal | Web interface for managing AI projects and assets. |
| Foundry SDK | SDK for building and automating Foundry-based solutions. |
| Foundry resource | Azure resource that provides compute, storage, tools, and services. |
| Foundry project | Project container for solution assets. |
| Model | A deployed AI model asset. |
| Agent | AI configuration with LLM, instructions, and tools. |
| Foundry IQ | Central knowledge connection capability in a Foundry project. |

### Lesson Flow

#### 1. Foundry as a Platform

Explain that Microsoft Foundry helps organize AI project assets and resources for Azure AI development.

#### 2. Resources and Projects

Explain that a Foundry resource can support one or more projects, and projects manage assets.

#### 3. Project Assets

Review models, agents, tools, and knowledge.

#### 4. Portal and SDK

Explain that developers can use the portal visually and the SDK programmatically.

### Suggested Activities

- Learners map Foundry project assets to examples.
- Learners decide whether a task is portal-based or SDK-based.

### Checks for Understanding

1. What is Microsoft Foundry?
2. What is a Foundry project?
3. Which assets can a project manage?
4. What is the difference between the portal and SDK?
