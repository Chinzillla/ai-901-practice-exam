# Module 11: Optimize Generative AI Model Performance with Microsoft Foundry

## Unit 2: Optimize model output with prompt engineering

### Source

[Microsoft Learn unit](https://learn.microsoft.com/en-us/training/modules/optimize-generative-ai-model-performance/2-prompt-engineering)

### Unit Overview

This unit teaches prompt engineering as the most accessible optimization strategy, including system messages, user and assistant messages, examples, delimiters, and generation parameters.

### Learning Objectives

- define prompt engineering
- identify prompt components
- write effective system message goals
- compare common prompt patterns
- explain temperature and top_p

### Key Terms

| Term | Beginner-Friendly Definition |
| --- | --- |
| System message | Instructions that define model role, behavior, boundaries, and output constraints. |
| Few-shot learning | Providing examples so the model follows a desired pattern. |
| Zero-shot learning | Asking the model to perform a task without examples. |
| Delimiter | A clear separator that distinguishes instructions, source text, and examples. |
| Temperature | A setting controlling randomness and creativity. |
| top_p | A setting limiting generation to a subset of probable next tokens. |

### Lesson Flow

#### 1. Prompt components

- System, user, assistant messages, and examples can shape model output.
- System messages can set role, boundaries, output format, and when-unsure behavior.

#### 2. Patterns

- Persona, format template, step-by-step decomposition, and few-shot examples improve reliability.
- Clear delimiters reduce confusion between instructions and content.

#### 3. Parameters

- Lower temperature supports factual tasks; higher temperature supports creative tasks.
- Usually adjust temperature or top_p, not both at once.

### Suggested Activities

- Ask learners to summarize the unit in one sentence using beginner-friendly language.
- Give learners a short workplace scenario and have them choose the correct concept, tool, strategy, or responsibility.
- Have learners identify one exam clue word from the unit, such as current, uploaded, benchmark, grounded, fine-tuned, guardrail, or endpoint.
- Pair learners to explain why one tempting alternative is not the best answer.

### Checks for Understanding

- What problem does this unit's main concept solve?
- Which keyword in a scenario would point you toward this concept?
- What is one limitation or caution learners should remember?
- How would this concept appear in a simple AI-901-style business scenario?

### Assessment

Use the accompanying question bank to check recognition, scenario selection, vocabulary, and common misconceptions. Learners should be able to explain not only the correct answer, but why the distractors are less appropriate.

### Teaching Notes

- Keep the explanation practical and scenario-based.
- Emphasize the difference between similar concepts instead of asking learners to memorize product names only.
- Connect the unit to AI-901 readiness by asking, "What would the exam scenario be asking me to choose?"
- Reinforce that secure, responsible, and measured development choices matter in generative AI apps.
