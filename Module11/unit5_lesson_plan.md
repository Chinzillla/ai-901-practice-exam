# Module 11: Optimize Generative AI Model Performance with Microsoft Foundry

## Unit 5: Compare and combine optimization strategies

### Source

[Microsoft Learn unit](https://learn.microsoft.com/en-us/training/modules/optimize-generative-ai-model-performance/5-compare-combine-strategies)

### Unit Overview

Learners compare prompt engineering, RAG, and fine-tuning. The unit emphasizes that they solve different problems and can be layered together.

### Learning Objectives

- compare strategy trade-offs
- identify when to use prompt engineering
- identify when to add RAG
- identify when to add fine-tuning
- choose combined strategies for a scenario

### Key Terms

| Term | Beginner-Friendly Definition |
| --- | --- |
| Optimization spectrum | The range from quick prompt changes to retrieval grounding and model fine-tuning. |
| Context optimization | Improving factual accuracy by providing relevant data, often through RAG. |
| Model optimization | Improving consistent behavior by changing model weights through fine-tuning. |
| Incremental approach | Starting simple and adding complexity only when needed. |

### Lesson Flow

#### 1. Strategy roles

- Prompt engineering guides behavior and is the starting point.
- RAG supplies data the model needs to know.
- Fine-tuning improves consistent style, tone, format, or tool behavior.

#### 2. Combinations

- Prompt engineering plus RAG handles behavior and factual context.
- RAG plus fine-tuning handles grounding and consistent behavior.
- All three can be combined for demanding apps.

#### 3. Decision framework

- Start with prompts, add RAG for accuracy, add fine-tuning for consistency, and combine as needed.

### Suggested Activities

- Ask learners to summarize the unit in one sentence using beginner-friendly language.
- Give learners a short workplace scenario and have them choose the correct concept, tool, strategy, or responsibility.
- Have learners identify one exam clue word from the unit, such as current, uploaded, benchmark, grounded, fine-tuned, guardrail, or endpoint.
- Pair learners to explain why one tempting alternative is not the best answer.

### Checks for Understanding

- What problem does this unit's main concept solve?
- Which keyword in a scenario would point you toward this concept?
- What is one limitation or caution learners should remember?
- How would this concept appear in a simple AI-901-style business scenario?

### Assessment

Use the accompanying question bank to check recognition, scenario selection, vocabulary, and common misconceptions. Learners should be able to explain not only the correct answer, but why the distractors are less appropriate.

### Teaching Notes

- Keep the explanation practical and scenario-based.
- Emphasize the difference between similar concepts instead of asking learners to memorize product names only.
- Connect the unit to AI-901 readiness by asking, "What would the exam scenario be asking me to choose?"
- Reinforce that secure, responsible, and measured development choices matter in generative AI apps.
