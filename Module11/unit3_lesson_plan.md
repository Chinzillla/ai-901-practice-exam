# Module 11: Optimize Generative AI Model Performance with Microsoft Foundry

## Unit 3: Ground your model with Retrieval Augmented Generation

### Source

[Microsoft Learn unit](https://learn.microsoft.com/en-us/training/modules/optimize-generative-ai-model-performance/3-retrieval-augmented-generation)

### Unit Overview

Learners study RAG as the main approach for grounding a model with current, private, or domain-specific data. The unit includes embeddings, vector search, Azure AI Search, and hybrid search.

### Learning Objectives

- define grounding and RAG
- describe retrieve-augment-generate steps
- explain embeddings and vector search
- identify when Azure AI Search supports RAG
- recognize when RAG is appropriate

### Key Terms

| Term | Beginner-Friendly Definition |
| --- | --- |
| Grounding | Providing trusted context so the model bases output on relevant facts. |
| RAG | Retrieval Augmented Generation: retrieve relevant data, add it to the prompt, and generate a grounded answer. |
| Embedding | A vector representation of text meaning. |
| Vector search | Searching by semantic similarity using embeddings. |
| Hybrid search | Combining keyword, semantic, and vector search for stronger retrieval. |
| Azure AI Search | An Azure service commonly used as the retrieval component for RAG solutions. |

### Lesson Flow

#### 1. Problem

- Models may not know private data, recent facts, or domain-specific content.
- Ungrounded answers can sound plausible but be wrong.

#### 2. RAG steps

- Retrieve relevant information, augment the prompt with context, and generate a response based on that context.

#### 3. Search techniques

- Keyword, semantic, vector, and hybrid search can support retrieval.
- Hybrid search is often recommended for generative AI apps.
