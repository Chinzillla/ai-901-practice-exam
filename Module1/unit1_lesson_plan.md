# Module 1: Introduction to AI Concepts

## Unit 1: Introduction to AI

### Source

Microsoft Learn: [Introduction to AI](https://learn.microsoft.com/en-us/training/modules/get-started-ai-fundamentals/1-introduction)

### Unit Overview

This unit introduces learners to the purpose of studying artificial intelligence and sets expectations for the module. It frames AI as a broad set of capabilities that can support many kinds of solutions, while keeping the focus at a beginner-friendly conceptual level rather than code or mathematical model details.

### Learning Objectives

By the end of this unit, learners should be able to:

- Describe the purpose of an introductory AI concepts module.
- Recognize that AI includes multiple capabilities and solution types.
- Explain why a high-level conceptual foundation matters before studying specific AI workloads.
- Distinguish between conceptual AI learning and technical implementation detail.
- Identify the role of text-based learning material when studying Microsoft Learn modules.

### Key Terms

| Term | Definition |
| --- | --- |
| Artificial intelligence (AI) | Software capabilities that enable systems to perform tasks that typically require human-like understanding, reasoning, perception, or decision support. |
| AI workload | A category of AI capability, such as generative AI, natural language processing, speech, computer vision, information extraction, or responsible AI. |
| Conceptual understanding | Knowledge of what a technology is, what it can do, and when it should be used. |
| Technical implementation | The practical details of building, configuring, coding, or deploying a solution. |
| Text-based learning | A learning format that presents concepts through written explanations and images rather than video. |

### Lesson Flow

#### 1. Why Learn AI Concepts

Start by explaining that many learners encounter AI through media, school, work, or new tools being adopted by organizations. The purpose of this unit is to create a foundation before moving into specific AI workloads.

Key teaching point: AI-901 expects learners to recognize concepts and match AI capabilities to scenarios, so understanding the vocabulary matters.

#### 2. AI as a Broad Field

Explain that AI is not one single feature. It includes several kinds of capabilities, such as generating content, analyzing language, recognizing speech, interpreting images, extracting information, and supporting responsible decision-making.

Use a simple scenario: an organization might use AI to summarize documents, answer customer questions, read invoices, describe images, or help users interact with applications through speech.

#### 3. Conceptual Learning Before Implementation

Clarify that this opening module is not focused on writing code or studying the mathematics behind machine learning models. Instead, it helps learners understand what AI can do and how common AI capabilities are categorized.

Key teaching point: before choosing or building an AI solution, learners should be able to identify the problem type and the appropriate AI workload.

#### 4. Using Text-Based Microsoft Learn Units

Explain that Microsoft Learn modules may offer video and text formats. For this course build, use the text version as the source because it generally provides more detail and is easier to reference when creating lessons and question banks.

Key teaching point: learners can use videos for quick orientation, but the text content is useful for careful review and exam preparation.

### Suggested Activities

#### Activity 1: AI Capability Brainstorm

Ask learners to list AI-powered tools they have seen at work, school, or online. Then classify each example into a broad capability such as language, speech, vision, generative AI, or information extraction.

#### Activity 2: Concept or Implementation

Give learners statements and ask whether each one is conceptual or implementation-focused.

Examples:

- "AI can summarize long documents." Conceptual.
- "Use a Python SDK to call a model endpoint." Implementation.
- "Computer vision analyzes images." Conceptual.
- "Configure authentication for an Azure resource." Implementation.

#### Activity 3: Exam Readiness Discussion

Ask learners why it is useful to understand categories of AI workloads before learning specific tools. Guide them toward the idea that exam questions often describe a scenario and expect the learner to choose the right capability.

### Checks for Understanding

Use these questions during or after instruction:

1. Why is it useful to learn AI concepts before implementation details?
2. What are examples of common AI workloads?
3. What is the difference between conceptual understanding and technical implementation?
4. Why might text-based learning material be useful for exam preparation?
5. How might AI-901 test a learner's understanding of AI workloads?

### Assessment

Use the accompanying `unit1_question_bank.md` file to assess learner understanding. The question bank includes multiple-choice and true-or-false questions with answers and rationales.

### Teaching Notes

- Keep this unit high level and welcoming for newcomers.
- Emphasize that AI-901 is not only about memorizing definitions; it also tests whether learners can identify the correct AI capability for a scenario.
- Avoid going deep into math, model training, or SDK details in this opening unit.
- Preview upcoming concepts such as generative AI, natural language, speech, computer vision, information extraction, and responsible AI.
