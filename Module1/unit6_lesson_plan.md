# Module 1: Introduction to AI Concepts

## Unit 6: Information Extraction

### Source

Microsoft Learn: [Information extraction](https://learn.microsoft.com/en-us/training/modules/get-started-ai-fundamentals/6-extract-insights)

### Unit Overview

This unit introduces AI-powered information extraction. Learners explore how AI can find useful information in unstructured sources such as scanned documents, forms, images, audio recordings, and video recordings. The unit emphasizes optical character recognition (OCR), document analysis, and business scenarios where extracted data becomes useful.

### Learning Objectives

By the end of this unit, learners should be able to:

- Define information extraction.
- Identify unstructured data sources that AI can analyze.
- Explain the role of optical character recognition.
- Describe how document analysis can map extracted text to fields.
- Recognize common information extraction scenarios.
- Choose information extraction capabilities for business requirements.

### Key Terms

| Term | Definition |
| --- | --- |
| Information extraction | Using AI to find and extract useful data values or insights from unstructured sources. |
| Unstructured data | Data that is not already organized in a fixed table or schema, such as scanned forms, images, audio, or video. |
| Optical character recognition (OCR) | Computer vision technology that identifies text in images. |
| Document analysis | Extracting and interpreting information from documents or forms. |
| Field extraction | Mapping extracted text to specific data fields, such as receipt total or invoice date. |
| Indexing | Organizing extracted content so documents can be searched. |

### Lesson Flow

#### 1. Introduce Information Extraction

Explain that organizations often have valuable information trapped in unstructured content. AI can help find and extract that information from documents, images, forms, audio, and video.

#### 2. OCR as a Foundation

Define OCR as a computer vision technology that identifies text in images. OCR is often the first step when the source is a scanned document, receipt, form, or photo containing text.

Key teaching point: OCR finds text; additional analysis may be needed to understand what the text means.

#### 3. From Text to Fields

Explain that document analysis can interpret extracted text and map it to fields. For example, text on a receipt can be mapped to merchant, date, total, and expense category fields.

#### 4. Beyond Text-Based Forms

Explain that newer extraction scenarios can include audio, images, and video. For example, a meeting transcript can be analyzed to identify key points and follow-up actions.

#### 5. Common Scenarios

Review common uses:

- Processing expense claims.
- Digitizing paper forms.
- Indexing documents for search.
- Identifying key points and follow-up actions from meetings.

### Suggested Activities

#### Activity 1: Source or Field

Give learners examples and ask them to identify the source type and likely extracted fields.

Example: a receipt image could produce merchant, date, total, and tax.

#### Activity 2: OCR or Field Extraction

Ask learners to distinguish between locating text in an image and mapping that text to business fields.

#### Activity 3: Business Process Mapping

Ask learners to choose one document-heavy business process and identify where information extraction could reduce manual entry.

### Checks for Understanding

1. What is information extraction?
2. What kinds of sources can contain unstructured data?
3. What does OCR do?
4. Why is OCR not always enough by itself?
5. What is field extraction?
6. Which scenarios commonly use information extraction?
