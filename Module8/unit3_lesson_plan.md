# Module 8: Select, Deploy, and Evaluate Microsoft Foundry Models

## Unit 3: Select models using benchmarks

### Source

[Microsoft Learn unit](https://learn.microsoft.com/en-us/training/modules/model-catalog-evaluate/3-select-models-benchmarks)

### Unit Overview

Learners use benchmarks to compare model candidates. The unit explains that benchmarks can measure quality, safety, cost, latency, throughput, and other trade-offs.

### Learning Objectives

- identify common benchmark categories
- explain why benchmark trade-offs matter
- compare latency and throughput
- choose a model based on workload priorities

### Key Terms

| Term | Beginner-Friendly Definition |
| --- | --- |
| Quality benchmark | A test that evaluates task performance such as reasoning, coding, math, or instruction following. |
| Safety benchmark | A test that estimates risk for harmful, toxic, or unsafe outputs. |
| Latency | The time a model takes to begin or complete a response. |
| Throughput | The amount of work a model can process over time. |
| Token cost | The price associated with processing input and output tokens. |

### Lesson Flow

#### 1. Benchmark categories

- Quality, safety, cost, and performance benchmarks answer different selection questions.
- No single benchmark proves a model is best for every application.

#### 2. Performance metrics

- Latency metrics such as P50, P90, P95, and P99 help estimate response time for users.
- Throughput helps estimate how much traffic a deployment can support.

#### 3. Trade-off decisions

- A model with high quality can still be a poor choice if cost or latency does not fit the workload.

### Suggested Activities

- Ask learners to summarize the unit in one sentence using beginner-friendly language.
- Give learners a short workplace scenario and have them choose the correct concept, tool, strategy, or responsibility.
- Have learners identify one exam clue word from the unit, such as current, uploaded, benchmark, grounded, fine-tuned, guardrail, or endpoint.
- Pair learners to explain why one tempting alternative is not the best answer.

- What is one limitation or caution learners should remember?
- How would this concept appear in a simple AI-901-style business scenario?

### Assessment

Use the accompanying question bank to check recognition, scenario selection, vocabulary, and common misconceptions. Learners should be able to explain not only the correct answer, but why the distractors are less appropriate.

### Teaching Notes

- Keep the explanation practical and scenario-based.
- Emphasize the difference between similar concepts instead of asking learners to memorize product names only.
- Connect the unit to AI-901 readiness by asking, "What would the exam scenario be asking me to choose?"
- Reinforce that secure, responsible, and measured development choices matter in generative AI apps.
