# Module 8: Select, Deploy, and Evaluate Microsoft Foundry Models

## Unit 1: Introduction

### Source

[Microsoft Learn unit](https://learn.microsoft.com/en-us/training/modules/model-catalog-evaluate/1-introduction)

### Unit Overview

Learners begin the model selection workflow in Microsoft Foundry. The unit frames why model choice matters, how benchmarks help compare options, and how deployment and evaluation complete the model lifecycle.

### Learning Objectives

- describe the model selection workflow
- identify why benchmarks are useful
- recognize the relationship between deployment and evaluation
- explain why model choice affects quality, cost, and performance

### Key Terms

| Term | Beginner-Friendly Definition |
| --- | --- |
| Foundation model | A pretrained model that can be adapted or used as the base for generative AI applications. |
| Model catalog | A searchable collection of models available through Microsoft Foundry. |
| Benchmark | A measurement used to compare model quality, safety, cost, or performance. |
| Deployment | A hosted endpoint that makes a selected model available to an application. |
| Evaluation | The process of testing model output against quality, safety, and task requirements. |

### Lesson Flow

#### 1. Why selection matters

- Different models have different strengths, limits, cost profiles, and latency characteristics.
- A stronger model is not always the best model for every workload.

#### 2. The Foundry workflow

- Explore the catalog, compare candidates, deploy a model, test it, and evaluate the results.
- Evaluation should be repeated as prompts, data, or model choices change.

#### 3. Exam angle

- AI-901 questions often ask learners to choose the best stage or metric for a scenario.

### Suggested Activities

- Ask learners to summarize the unit in one sentence using beginner-friendly language.
- Give learners a short workplace scenario and have them choose the correct concept, tool, strategy, or responsibility.
- Have learners identify one exam clue word from the unit, such as current, uploaded, benchmark, grounded, fine-tuned, guardrail, or endpoint.
- Pair learners to explain why one tempting alternative is not the best answer.

### Checks for Understanding

- What problem does this unit's main concept solve?
- Which keyword in a scenario would point you toward this concept?
- What is one limitation or caution learners should remember?
- How would this concept appear in a simple AI-901-style business scenario?

### Assessment

Use the accompanying question bank to check recognition, scenario selection, vocabulary, and common misconceptions. Learners should be able to explain not only the correct answer, but why the distractors are less appropriate.

### Teaching Notes

- Keep the explanation practical and scenario-based.
- Emphasize the difference between similar concepts instead of asking learners to memorize product names only.
- Connect the unit to AI-901 readiness by asking, "What would the exam scenario be asking me to choose?"
- Reinforce that secure, responsible, and measured development choices matter in generative AI apps.
