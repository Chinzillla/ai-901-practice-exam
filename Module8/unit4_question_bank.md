# Module 8: Select, Deploy, and Evaluate Microsoft Foundry Models

## Unit 4 Question Bank: Deploy models to endpoints

These questions are authored from the unit's Microsoft Learn concepts and rewritten as exam-style practice items. Answer choices are intentionally paraphrased instead of copied from the lesson text.

---

### 1. A model has been selected and must be made callable by an app. What is the next step?

A. deployment  
B. configuration  
C. authentication  
D. Deploy the model to an endpoint  

**Answer:** D  
**Rationale:** The requirement matches the unit capability named in the correct option.

---

### 2. What is the main purpose of "deployment"?

A. settings such as model choice, capacity, or access details for a deployment  
B. verifying that callers are allowed to use the deployed endpoint  
C. making a selected model available for application use  
D. the URL or service target an app calls to use a deployed model  

**Answer:** C  
**Rationale:** "deployment" is a core idea in this unit, and the correct option paraphrases its role.

---

### 3. A client app needs the address for sending model requests. What value does it need?

A. Speech synthesis  
B. The endpoint  
C. Prompt engineering  
D. Object detection  

**Answer:** B  
**Rationale:** The correct option is the unit concept that best fits the stated requirement.

---

### 4. How is selecting a model different from deploying it?

A. Selection chooses the model; deployment makes it available for calls from apps.  
B. Deployment only reads documentation.  
C. Selection automatically creates every endpoint.  
D. They are unrelated to application integration.  

**Answer:** A  
**Rationale:** The unit separates choosing models from creating usable deployments.

---

### 5. What should a learner remember about "endpoint"?

A. a configuration task for the interface rather than the AI behavior  
B. a rule-based process that does not use model reasoning  
C. an access-control setting rather than the concept being defined  
D. URL or service target an app calls to use a deployed model  

**Answer:** D  
**Rationale:** "endpoint" is tested by recognizing its purpose, not by memorizing wording from the page.

---

### 6. Why does endpoint configuration matter?

A. Configuration prevents authentication.  
B. All deployments have identical behavior.  
C. It affects how apps access the model and how the deployment supports expected workload.  
D. Configuration only changes font color.  

**Answer:** C  
**Rationale:** Deployment settings influence access and runtime behavior.

---

### 7. Which two items does an app commonly need to call a deployed model? Choose two.

A. Client display size  
B. Image caption only  
C. local file path unrelated to the endpoint  
D. Endpoint URL  
E. Credential or access token  

**Answer:** D, E  
**Rationale:** Application calls require a target endpoint and authorized access.

---

### 8. A developer controls which applications can call a deployed model. Which concern is this?

A. Authentication and access control  
B. endpoint  
C. Information extraction  
D. Model evaluation  

**Answer:** A  
**Rationale:** The scenario points to the unit capability described by the correct option.

---

### 9. Which three activities are part of model deployment work? Choose three.

A. Ignore access control  
B. Assume no monitoring is needed  
C. Manually type every response  
D. Choose deployment settings  
E. Create or select an endpoint  
F. Test calls against the deployment  

**Answer:** D, E, F  
**Rationale:** Deployment includes configuration, endpoint access, and testing.

### 10. Which description best matches "configuration"?

C. configuration values such as model choice, capacity, or access details for a deployment  
D. Choose a model based on popularity instead of task fit.  

**Answer:** C  
**Rationale:** The correct answer paraphrases how the unit explains "configuration".
