# Module 8: Select, Deploy, and Evaluate Microsoft Foundry Models

## Unit 5: Evaluate model performance

### Source

[Microsoft Learn unit](https://learn.microsoft.com/en-us/training/modules/model-catalog-evaluate/5-evaluate-performance)

### Unit Overview

Learners study manual and automated evaluation approaches for generative AI models. The unit connects evaluation to quality, safety, compliance, user satisfaction, and continuous improvement.

### Learning Objectives

- compare manual and automated evaluation
- identify quality and safety evaluation metrics
- explain why evaluation is iterative
- choose an evaluation approach for a scenario

### Key Terms

| Term | Beginner-Friendly Definition |
| --- | --- |
| Manual evaluation | Human review of model outputs against criteria such as relevance, accuracy, and safety. |
| Automated evaluation | Programmatic scoring of model outputs using evaluators or metrics. |
| Groundedness | The degree to which a response is supported by provided source data. |
| Relevance | How well a response addresses the user's prompt. |
| Defect rate | A safety measurement that estimates how often outputs violate defined criteria. |

### Lesson Flow

#### 1. Evaluation reasons

- Evaluation helps assess quality, safety, compliance, and user experience.
- It creates evidence for model selection and improvement.

#### 2. Manual review

- Reviewers can score relevance, informativeness, engagement, accuracy, and safety.
- Manual evaluation is useful early or for nuanced requirements.

#### 3. Automated metrics

- Quality metrics can include groundedness, relevance, coherence, and fluency.
- Safety metrics can track categories such as hateful content, violence, self-harm, sexual content, and jailbreak attempts.

### Teaching Notes

- Keep the explanation practical and scenario-based.
- Emphasize the difference between similar concepts instead of asking learners to memorize product names only.
- Connect the unit to AI-901 readiness by asking, "What would the exam scenario be asking me to choose?"
- Reinforce that secure, responsible, and measured development choices matter in generative AI apps.
