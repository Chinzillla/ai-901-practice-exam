### 1. A team compares models using published scores on a reasoning task. What are they using?

### 2. What is the main purpose of "benchmark"?

A. the time a model takes to return a response  
B. the amount of work a model can handle over time  
C. a standardized way to compare model performance on known tasks  
D. a score that reflects how well a model performs for a measured task  

**Answer:** C  
**Rationale:** "benchmark" is a core idea in this unit, and the correct option paraphrases its role.

### 3. A chatbot must respond quickly during live conversations. Which benchmark dimension matters most?

### 4. Why are benchmark scores not the only factor in model selection?

A. A model also needs to match the real task, cost, latency, and deployment requirements.  
B. Benchmarks prove a model is safe for every use.  
C. Benchmark scores replace testing with your own prompts.  
D. A higher score always means lower cost.  

**Answer:** A  
**Rationale:** The unit positions benchmarks as useful evidence, not the entire decision.

### 5. What should a learner remember about "accuracy metric"?

C. an access-control setting rather than the concept being defined

### 6. How should a developer use benchmark results?

A. Choose the slowest model when accuracy is irrelevant.  
B. Ignore the actual workload.  
C. Use them to shortlist models, then validate with the solution's own scenarios.  
D. Use them to avoid all evaluation.  

**Answer:** C  
**Rationale:** Benchmarks guide selection but should be combined with scenario testing.

### 9. Which three factors should be balanced when selecting a model using benchmarks? Choose three.
