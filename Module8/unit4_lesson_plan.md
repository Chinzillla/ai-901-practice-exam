# Module 8: Select, Deploy, and Evaluate Microsoft Foundry Models

## Unit 4: Deploy models to endpoints

### Source

[Microsoft Learn unit](https://learn.microsoft.com/en-us/training/modules/model-catalog-evaluate/4-deploy-models)

### Unit Overview

This unit explains how selected models are deployed to endpoints in Microsoft Foundry and how developers test and connect to deployments.

### Learning Objectives

- describe why models are deployed
- identify common deployment information needed by apps
- recognize deployment type considerations
- explain how the playground supports testing

### Key Terms

| Term | Beginner-Friendly Definition |
| --- | --- |
| Endpoint | A hosted access point that an application calls to use a deployed model. |
| Deployment name | The identifier an application uses to target a specific deployed model. |
| Authentication key | A secret used by an application to authenticate to a model endpoint. |
| Global Standard | A deployment option commonly recommended when available for broad capacity needs. |
| Playground | An interactive Foundry experience for testing a deployed model before coding. |

### Lesson Flow

#### 1. Deployment purpose

- Deployment turns a catalog model into a callable service.
- Applications need endpoint details and authentication to send prompts.

#### 2. Deployment choices

- Options can include global, regional, provisioned, batch, data-zone, and developer deployment types.
- The right choice depends on availability, cost, throughput, and governance needs.

#### 3. Post-deployment testing

- The playground helps learners test behavior before integrating the endpoint into code.
