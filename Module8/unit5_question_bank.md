### 1. A team tests a model with realistic prompts before releasing it. What activity is this?

**Rationale:** This scenario matches Evaluate model performance as described in the unit.

### 2. What is the main purpose of "evaluation"?

A. a measurement that reflects how useful or correct model output is  
B. checking whether a model produces harmful, unsafe, or inappropriate outputs  
C. measuring how well a model performs for intended tasks

**Answer:** C  
**Rationale:** "evaluation" is a core idea in this unit, and the correct option paraphrases its role.

### 3. A model is checked for harmful responses before production use. Which evaluation area is involved?

### 4. Why should evaluation use examples that match the real solution?

A. Realistic examples reveal whether the model performs well for the intended workload.  
B. Unrelated examples always prove production readiness.  
C. Evaluation is only needed for image models.

**Rationale:** Evaluation should reflect actual use cases.

### 6. What should happen if evaluation reveals weak or unsafe outputs?

A. Delete all evaluation records.  
B. Assume users will not notice problems.  
C. Adjust prompts, model choice, grounding, or safeguards and test again.

**Answer:** C  
**Rationale:** Evaluation informs iteration and risk reduction.
