# Module 6: Introduction to AI-Powered Information Extraction Concepts

## Unit 5: Exercise - Explore AI Information Extraction

### Source

Microsoft Learn: [Exercise - Explore AI information extraction](https://learn.microsoft.com/en-us/training/modules/introduction-information-extraction/6b-exercise)

### Unit Overview

This exercise unit asks learners to explore OCR combined with a large language model to extract and interpret fields from receipts.

### Learning Objectives

- Observe OCR extracting text from receipts.
- Observe how an LLM can interpret fields.
- Map extracted values to a business schema.
- Reflect on confidence, validation, and review needs.

### Key Terms

| Term | Definition |
| --- | --- |
| Receipt extraction | Extracting values such as merchant, date, items, tax, and total. |
| Schema | The target structure for extracted fields. |
| Validation | Checking extracted values for correctness. |

### Lesson Flow

#### 1. Exercise Orientation

Explain that learners should observe both OCR and semantic field interpretation.

#### 2. Field Mapping

Have learners identify receipt values and their target fields.

#### 3. Review and Validation

Discuss how confidence and validation checks can reduce errors.

### Suggested Activities

- Learners list extracted fields from a sample receipt.
- Learners identify one validation check for receipt totals.

### Checks for Understanding

1. What does OCR contribute?
2. What does the LLM contribute?
3. Which receipt fields were extracted?
4. Why might validation be required?
