# Module 6: Introduction to AI-Powered Information Extraction Concepts

## Unit 2: Overview of Information Extraction

### Source

Microsoft Learn: [Overview of information extraction](https://learn.microsoft.com/en-us/training/modules/introduction-information-extraction/2-overview)

### Unit Overview

This unit explains information extraction as a workload combining computer vision, OCR, machine learning, and generative AI to detect text and map values to data fields.

### Learning Objectives

- Explain the two broad steps of information extraction.
- Describe OCR's role.
- Explain value identification and field mapping.
- Identify planning considerations for extraction solutions.

### Key Terms

| Term | Definition |
| --- | --- |
| OCR | Optical character recognition, used to convert images of text to machine-readable text. |
| Field mapping | Associating extracted text values with business fields. |
| Human-in-the-loop | Human validation or review in an automated process. |
| Latency | How quickly processing must complete. |

### Lesson Flow

#### 1. Extraction Workflow

Explain OCR followed by value identification and field mapping.

#### 2. Example: Expense Claims

Use receipt processing to show vendor, date, subtotal, tax, and total extraction.

#### 3. Planning Considerations

Review document layout consistency, volume, accuracy, security, processing power, latency, scalability, and integration complexity.

### Suggested Activities

- Learners choose template-based or ML-based extraction for different document sets.
- Learners identify constraints for a high-volume invoice processing system.

### Checks for Understanding

1. What role does OCR play?
2. What does field mapping do?
3. Why might human validation be needed?
