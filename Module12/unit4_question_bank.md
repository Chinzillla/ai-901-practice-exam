# Module 12: Implement a Responsible Generative AI Solution in Microsoft Foundry

## Unit 4 Question Bank: Measure potential harms

These questions are authored from the unit's Microsoft Learn concepts and rewritten as exam-style practice items. Answer choices are intentionally paraphrased instead of copied from the lesson text.

---

### 1. A team runs risky prompts against a model and records how often unsafe answers appear. What are they doing?

A. harm measurement  
B. baseline  
C. test dataset  
D. Measuring potential harms  

**Answer:** D  
**Rationale:** The requirement matches the unit capability named in the correct option.

---

### 2. What is the main purpose of "harm measurement"?

A. examples used to evaluate risky or unwanted model behavior  
B. a defined measure for judging harm, quality, or safety  
C. testing and evaluating how often or severely potential harms appear  
D. an initial measurement used for comparison after changes  

**Answer:** C  
**Rationale:** "harm measurement" is a core idea in this unit, and the correct option paraphrases its role.

---

### 3. Before adding safeguards, a team records the current unsafe-response rate. What is that first measurement?

A. Object detection  
B. Baseline capability  
C. evaluation metric  
D. Prompt engineering  

**Answer:** B  
**Rationale:** The correct option is the unit concept that best fits the stated requirement.

---

### 4. Why is measurement needed after mapping harms?

A. Measurement shows how often or severely identified risks occur so improvements can be compared.  
B. Mapping harms automatically fixes them.  
C. Measurement is only for billing.  
D. Measurement prevents the use of test data.  

**Answer:** A  
**Rationale:** The unit moves from identifying harms to quantifying them.

### 5. What should a learner remember about "baseline"?

A. a configuration task for the interface rather than the AI behavior  
B. a rule-based process that does not use model reasoning  
C. an access-control setting rather than the concept being defined  
D. initial measurement used for comparison after changes  

**Answer:** D  
**Rationale:** "baseline" is tested by recognizing its purpose, not by memorizing wording from the page.

### 6. Why define evaluation metrics before comparing mitigations?

A. Metrics only affect UI colors.  
B. Mitigations cannot be compared.  
C. Metrics create a consistent way to judge whether risk has been reduced.  
D. Metrics should change randomly during testing.  

**Answer:** C  
**Rationale:** Consistent measurement supports responsible iteration.

---

### 7. Which two items support harm measurement? Choose two.

A. Unrelated interface preference  
B. Hidden credentials in prompts  
C. Ignoring test failures  
D. Baseline results  
E. Evaluation metrics  

**Answer:** D, E  
**Rationale:** A baseline and metrics allow teams to compare risk before and after changes.

---

### 8. A set of example prompts is used to test for harmful content. What is it?

A. Test dataset capability  
B. Speech synthesis  
C. Information extraction  
D. Model evaluation  

**Answer:** A  
**Rationale:** The scenario points to the unit capability described by the correct option.

---

### 9. Which three practices belong in measuring potential harms? Choose three.

A. Delete failed tests  
B. Assume no harm is possible  
C. Avoid stakeholder review  
D. Use representative test prompts  
E. Record harmful output rates  
F. Compare results over time  

**Answer:** D, E, F  
**Rationale:** Measurement requires examples, results, and comparison.

### 10. Which description best matches "test dataset"?

A. Rely on one successful trial instead of broader evaluation.  
B. Use a visual analysis capability when the input is only text.  
C. test cases used to evaluate risky or unwanted model behavior  
D. Choose a model based on popularity instead of task fit.  

**Answer:** C  
**Rationale:** The correct answer paraphrases how the unit explains "test dataset".
